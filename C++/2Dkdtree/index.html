2D k-d tree
General information can be found on wikipedia.

Features of my implementation:

• Tree can be constructed in balanced configuration if all points are known beforehand.

• Dedicated implementation of point relocation. Relocation is always possible to simulate via deletion and reinsertion of a point. However, such process is quite inefficient. My implementation uses two approaches to speed up the process:

- Memory recycling - k-d tree uses internal structures that have to be allocated/freed when new point is inserted/deleted. So for relocation, pointers to already allocated memory are stored and reused.

- Bottom up searching for new subtree - when points are relocated, the new location is usually not far from the original location and this can be exploited. If point represents leaf node and is located within its bounding rectangle after relocation, it is only necessary to change its coordinates (tree structure stays the same). If it moved over the bounding rectangle or is not a leaf node, the new position is most likely going to be in a subtree with the same parent or grandparent. So from the subtree that the point is located in, the function starts moving upwards and in each parent, it checks whether or not the point belongs to the bounding rectangle of that subtree.

• Searching for arbitrary number of nearest neighbours is possible as well as searching for n-th closest neighbour.
